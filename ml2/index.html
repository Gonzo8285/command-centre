SharePoint reclaim & automation delivering

ClaudeBridge — Multi-agent operating protocol infrastructure

File-bus on Google Drive · per-seat inbox/outbox · ACK windows · heartbeat cadence

ML²'s own operating-model infrastructure, productised. ClaudeBridge is the protocol that lets multiple AI agents on different machines coordinate on a single portfolio: per-seat inboxes, priority-tiered ACK windows (P0=10min through P3=none), heartbeat liveness, and a verification handshake. It's the difference between agents-as-chatbots and agents-as-operations-layer.

  • v3 layout: per-seat nodes/<seat>/{inbox,outbox,status,memory}/
  • Truth-source rule: every "deployed" claim verified against the live public URL
  • Bidirectional verification handshake; mirrored memory; cron-fallback heartbeats
  • Available as a reference architecture for clients running their own multi-agent deployments
